Transmissor Analógico (4-20 mA)

Mapeamento de Sinais Elétricos para Grandezas de Engenharia

Disciplina: Medição de Vazão Sinal: 4 a 20 mA Faixa: 0 a 150 m³/h Plataforma: Arduino + Tinkercad

🎯 1. Objetivos de Aprendizagem

Ao final desta prática, o aluno será capaz de:
  • Compreender o padrão industrial de transmissão de sinal analógico 4-20 mA.
  • Mapear valores lidos do Conversor Analógico-Digital (ADC) para uma escala de corrente.
  • Converter o sinal elétrico (mA) para a grandeza física correspondente (Vazão em m³/h).
  • Implementar uma função de mapeamento linear preciso (float) no Arduino, superando as limitações da função map() padrão.

📖 2. Fundamentos Teóricos

2.1 O Padrão 4-20 mA

Na indústria, sensores e transmissores raramente enviam o valor direto da medição (ex: "100 m³/h"). Em vez disso, eles convertem a medição física em um sinal elétrico proporcional. O padrão mais comum é o loop de corrente de 4 a 20 mA.

  • 4 mA (Zero Vivo): Representa o limite inferior da faixa (ex: 0 m³/h). Diferente de 0 mA, o 4 mA permite ao sistema identificar se o cabo rompeu (se cair para 0 mA, é falha, não zero de vazão).
  • 20 mA: Representa o limite superior da faixa calibrada (ex: 150 m³/h).

2.2 Leitura pelo CLP / Microcontrolador

Microcontroladores como o Arduino leem tensão, não corrente. Portanto, nos painéis de controle, coloca-se um resistor de precisão (geralmente de 250 Ω) no circuito. Pela Lei de Ohm (V = R × I):

  • 4 mA × 250 Ω = 1 V
  • 20 mA × 250 Ω = 5 V

O Arduino Uno possui um ADC de 10 bits que converte tensões de 0 a 5V em valores matemáticos de 0 a 1023. Nesta simulação, usaremos um potenciômetro ligado de 0 a 5V para simular diretamente o sinal condicionado que chega ao pino A0 do Arduino.

2.3 Equação de Mapeamento (Regra de Três Linear)

Para relacionar as escalas, utilizamos uma interpolação linear:

ValorOut = (ValorIn − In_Min) × (Out_Max − Out_Min) / (In_Max − In_Min) + Out_Min

💻 5. Código Arduino – Comentado

#include <LiquidCrystal.h>

// LCD 16x2 Paralelo — nativo no Tinkercad (RS, EN, D4, D5, D6, D7)
LiquidCrystal lcd(12, 11, 6, 5, 4, 3);

const int PIN_SENSOR = A0; // Potenciômetro simula o sinal 4-20 mA condicionado

// Função personalizada mapFloat() 
// O map() nativo só trabalha com inteiros. Para precisão decimal, usamos float.
float mapFloat(float x, float in_min, float in_max, float out_min, float out_max) {
  return (x - in_min) * (out_max - out_min) / (in_max - in_min) + out_min;
}

void setup() {
  lcd.begin(16, 2);
  lcd.setCursor(0, 0);
  lcd.print("Pratica 1:");
  lcd.setCursor(0, 1);
  lcd.print("Transmissor");
  delay(2000);
  lcd.clear();
}

void loop() {
  // 1. Lê o sinal do "sensor"
  int leituraADC = analogRead(PIN_SENSOR);

  // 2. Mapeia ADC (0 a 1023) -> Corrente (4.0 a 20.0 mA)
  float corrente_mA = mapFloat(leituraADC, 0, 1023, 4.0, 20.0);

  // 3. Mapeia Corrente (4.0 a 20.0 mA) -> Vazão (0.0 a 150.0 m³/h)
  float vazao_m3h = mapFloat(corrente_mA, 4.0, 20.0, 0.0, 150.0);

  // 4. Exibe os resultados no LCD
  lcd.setCursor(0, 0);
  lcd.print("I: ");
  lcd.print(corrente_mA, 1);
  lcd.print(" mA    "); // espaços limpam dígitos anteriores
  
  lcd.setCursor(0, 1);
  lcd.print("Q: ");
  lcd.print(vazao_m3h, 1);
  lcd.print(" m3/h  ");
  
  delay(250);
}

📋 6. Procedimento Experimental

  1. Acesse o Tinkercad Circuits e inicie um novo projeto.
  2. Adicione um Arduino Uno, um Potenciômetro e um LCD 16×2 (display paralelo — sem módulo I2C). Todos estão na biblioteca padrão do Tinkercad.
  3. Faça as conexões de acordo com a Seção 4. O pino Wiper (central do potenciômetro) vai ao A0; o V0 do LCD vai ao GND (contraste máximo).
  4. Na aba Código, escolha Texto, apague o código padrão e cole o código da Seção 5.
  5. Clique em Iniciar Simulação.
  6. Gire o potenciômetro: observe I (corrente em mA) e Q (vazão em m³/h) atualizando no LCD conforme a tensão em A0 varia de 0 V a 5 V.
  7. Com o código rodando, responda às questões de análise.
